# 安全政策

## 报告安全漏洞

我们非常重视WPForge的安全性。如果您发现了安全漏洞，请负责任地披露，不要公开披露。

### 如何报告

**请通过以下方式报告安全漏洞：**

- **邮箱**: security@wpforge.dev
- **GitHub**: 提交 [Security Advisory](https://github.com/worldop123/wpforge/security/advisories/new)
- **PGP**: 如需加密通信，请联系我们获取公钥

**请在报告中包含以下信息：**
- 漏洞类型
- 受影响的版本
- 复现步骤
- 影响范围
- 可能的修复方案（如果有）

### 响应时间

我们承诺：
- **24小时内**确认收到您的报告
- **72小时内**初步评估漏洞严重程度
- **每周**更新修复进度
- **修复完成后**公开致谢（如您同意）

### 安全研究

我们欢迎安全研究人员的贡献。如果您：
- 遵守本安全政策
- 不访问或修改他人数据
- 不进行拒绝服务攻击
- 不泄露用户隐私
- 只测试您自己的系统

我们不会对您采取法律行动。

## 安全架构

### 认证与授权

- **JWT认证**: 无状态Token认证
- **密码哈希**: bcrypt算法，cost=12
- **权限分级**: 超级管理员/管理员/用户/访客
- **API密钥**: 支持应用专用密钥

### 数据安全

- **传输加密**: 强制HTTPS/TLS 1.3
- **存储加密**: 敏感字段加密存储
- **数据隔离**: 多租户数据隔离
- **备份加密**: 备份数据加密存储

### 输入验证

- **白名单验证**: 所有输入使用白名单验证
- **输出转义**: 所有输出自动转义
- **SQL注入防护**: ORM参数化查询
- **XSS防护**: 内容安全策略(CSP)
- **CSRF防护**: Token验证 + SameSite Cookie

### 网络安全

- **防火墙**: Web应用防火墙(WAF)
- **速率限制**: API请求频率限制
- **IP黑名单**: 恶意IP自动封禁
- **请求签名**: 关键请求签名验证

## 安全最佳实践

### 部署安全

1. **使用Docker部署**
   - 容器隔离
   - 非root用户运行
   - 只读文件系统

2. **网络隔离**
   - 数据库不暴露公网
   - Redis不暴露公网
   - 使用内部网络

3. **HTTPS强制**
   - 配置SSL证书
   - 强制HTTPS重定向
   - HSTS启用

### 配置安全

1. **强密码**
   - 管理员密码至少12位
   - 包含大小写字母、数字、特殊字符
   - 定期更换密码

2. **API密钥管理**
   - 不要硬编码密钥
   - 使用环境变量
   - 定期轮换密钥

3. **访问控制**
   - 最小权限原则
   - 定期审计用户权限
   - 禁用不用的账户

### 数据安全

1. **定期备份**
   - 每日自动备份
   - 异地备份
   - 定期测试恢复

2. **数据加密**
   - 敏感数据加密存储
   - 传输加密
   - 备份加密

3. **日志审计**
   - 记录关键操作
   - 定期审计日志
   - 异常告警

## 安全更新流程

### 漏洞处理流程

1. **接收报告**
   - 确认收到报告
   - 分配漏洞编号
   - 评估严重程度

2. **漏洞分析**
   - 复现漏洞
   - 分析根因
   - 评估影响范围

3. **修复开发**
   - 开发修复方案
   - 代码审查
   - 安全测试

4. **版本发布**
   - 准备安全更新
   - 更新CHANGELOG
   - 发布新版本

5. **公开披露**
   - 发布安全公告
   - 致谢报告者
   - 更新文档

### 严重程度分级

| 级别 | 描述 | 响应时间 |
|------|------|----------|
| 严重 | 远程代码执行、数据泄露 | 24小时内修复 |
| 高危 | 权限绕过、SQL注入 | 72小时内修复 |
| 中危 | XSS、CSRF、信息泄露 | 1周内修复 |
| 低危 | 轻微安全问题 | 下个版本修复 |

## 依赖安全

### 依赖管理

- **定期扫描**: 每周自动扫描依赖漏洞
- **及时更新**: 安全漏洞优先更新
- **锁定版本**: 使用lock文件确保一致性
- **最小依赖**: 只添加必要的依赖

### Python依赖

- 使用 `pip-audit` 扫描
- 定期更新依赖
- 关注安全公告

### JavaScript依赖

- 使用 `npm audit` 扫描
- 定期更新依赖
- 移除不用的依赖

### Docker镜像

- 使用官方基础镜像
- 定期更新基础镜像
- 扫描镜像漏洞
