# 贡献者行为准则

## 我们的承诺

为了营造开放和友好的环境，我们承诺让每个人参与我们的项目和社区都能获得无骚扰的体验，无论年龄、体型、残疾、种族、性别认同和表达、经验水平、国籍、个人外貌、种族、宗教或性取向如何。

## 我们的标准

### 积极的行为

有助于创造积极环境的行为包括：

- 使用友好和包容的语言
- 尊重不同的观点和经验
- 优雅地接受建设性批评
- 关注对社区最有利的事情
- 对其他社区成员表示同理心

### 不可接受的行为

参与者不可接受的行为包括：

- 使用性化的语言或图像，以及不受欢迎的性关注或骚扰
- 恶意评论、侮辱/贬损评论以及个人或政治攻击
- 公开或私下骚扰
- 未经明确许可发布他人的私人信息，如物理地址或电子邮件地址
- 在专业环境中可能被合理视为不当的其他行为

## 我们的责任

项目维护者有责任阐明可接受行为的标准，并应对任何不可接受的行为采取适当和公平的纠正措施。

## 适用范围

本行为准则适用于所有项目空间，也适用于个人在公共空间代表项目或其社区时。
代表项目或社区的示例包括使用官方项目电子邮件地址、通过官方社交媒体账户发布或在线上或线下活动中担任指定代表。
项目代表可以由项目维护者进一步定义和澄清。

## 执行

### 举报

如果您遇到或目睹了不可接受的行为，请通过以下方式联系项目团队：

- **邮箱**: conduct@wpforge.dev
- **GitHub**: 私信项目维护者

所有投诉都将被审查和调查，并将导致被认为必要和适合情况的回应。
项目团队有义务对事件的报告者保密。
具体执行政策的进一步细节可能会单独发布。

### 执行步骤

项目维护者将遵循以下社区影响准则，以确定对他们认为违反本行为准则的任何行为的后果：

#### 1. 纠正

**社区影响**: 使用不当语言或其他在社区中被认为不专业或不受欢迎的行为。

**后果**: 来自项目维护者的私人书面警告，明确说明违规的性质，并解释为什么该行为是不恰当的。
可能会要求公开道歉。

#### 2. 警告

**社区影响**: 通过单一事件或一系列行为违反。

**后果**: 对持续行为的警告。
在指定的时间段内，与相关人员没有互动，包括与执行行为准则的人主动互动。
这包括避免在社区空间以及社交媒体等外部渠道中互动。
违反这些条款可能会导致临时或永久禁令。

#### 3. 临时禁令

**社区影响**: 严重违反社区标准，包括持续的不当行为。

**后果**: 在指定的时间段内暂时禁止与社区进行任何形式的互动或公开交流。
在此期间，不允许与相关人员进行公开或私下互动，包括与执行行为准则的人主动互动。
违反这些条款可能会导致永久禁令。

#### 4. 永久禁令

**社区影响**: 展示出违反社区标准的行为模式，包括持续的不当行为、个人骚扰或对个人群体的攻击或贬低。

**后果**: 永久禁止在社区内进行任何形式的公开互动。

## 社区准则

### 沟通准则

- **保持尊重**: 即使存在分歧，也要保持尊重和专业的态度
- **建设性反馈**: 专注于想法和解决方案，而不是个人
- **耐心**: 对新人和不同技能水平的人要有耐心
- **清晰**: 尽量清晰简洁地沟通，考虑到语言可能不是每个人的母语

### 贡献准则

- **质量优先**: 提交高质量的代码，遵循项目的编码标准
- **测试**: 为您的代码编写测试，确保它不会破坏现有功能
- **文档**: 更新相关文档，包括代码注释和用户文档
- **小步提交**: 保持提交小而专注，便于审查

### 问题报告准则

- **详细描述**: 尽可能详细地描述问题，包括复现步骤
- **环境信息**: 包括您的操作系统、软件版本等环境信息
- **截图/日志**: 如果适用，包括截图或错误日志
- **搜索先**: 在提交新问题之前，搜索现有的问题以避免重复

### 拉取请求准则

- **描述清晰**: 清楚地描述您的更改和原因
- **关联问题**: 如果解决了某个问题，请关联该问题
- **测试通过**: 确保所有测试通过
- **代码审查**: 接受并回应代码审查反馈
- **一个PR一个功能**: 每个拉取请求只包含一个功能或修复

## 维护者职责

项目维护者有责任：

1. **维护秩序**: 确保社区保持友好和专业的氛围
2. **及时回应**: 在合理的时间内回应问题和拉取请求
3. **公平公正**: 公平公正地执行行为准则
4. **保密**: 对举报人的身份保密
5. **持续改进**: 不断改进社区和项目

## 常见问题

### Q: 如果我觉得自己被错误地指控了怎么办？

**A:** 如果您认为自己被错误地指控违反了行为准则，请按照举报流程进行回应。
项目团队将审查您的回应并做出公平的决定。

### Q: 行为准则适用于所有项目空间吗？

**A:** 是的，本行为准则适用于所有项目空间，包括但不限于：
- GitHub Issues 和 Pull Requests
- 项目聊天室和论坛
- 项目邮件列表
- 线下活动和聚会

### Q: 我可以匿名举报吗？

**A:** 是的，您可以匿名举报。
但是，请注意，匿名举报可能更难调查和处理。

### Q: 行为准则会更新吗？

**A:** 是的，行为准则可能会根据社区反馈和需求进行更新。
重大变更将通过项目渠道公布。
